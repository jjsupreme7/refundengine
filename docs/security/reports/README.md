# Security Audit Reports

This folder contains security audit reports, code reviews, and vulnerability assessments conducted on the Refund Engine system.

## Purpose

Security reports are archived to:
- Track security improvements over time
- Maintain audit trail for compliance
- Document vulnerabilities and their resolutions
- Provide historical context for security decisions

## Naming Convention

Security reports follow the naming convention:
```
SECURITY_REPORT_TYPE_YYYY-MM-DD.md
```

## Reporting Security Issues

If you discover a security vulnerability:
1. **Do not** create a public GitHub issue
2. Email details to the security team (see main README)
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fixes (if any)

## Adding New Reports

When adding security audit reports:

1. **Always include the date** in the filename
2. **Mark sensitivity level** at the top of the report
3. **Update this README** with report details
4. **Link to any related issues** or fixes implemented
